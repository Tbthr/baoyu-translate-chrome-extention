# 翻译指令

## 翻译风格

**storytelling** — 以引人入胜的叙事节奏进行翻译，像一位优秀的中文叙事者重新讲述这个故事。保持原文的幽默感、自嘲口吻和叙事张力，让中文读者获得与英文读者相同的阅读体验。

## 内容背景

本文是英国自由撰稿人 James O'Malley 发表在 Substack 上的新年个人随笔。文章以第一人称叙述，探讨了一个现代都市人的社交困境：生活被过度优化后，反而失去了与他人的真实连接。文章从个人经历出发，逐步上升到对社区、群体规范和人际连接的社会学思考。

目标读者：对生活方式、社会观察和个人成长感兴趣的中文读者。

## 翻译注意事项

1. **重写而非直译**：用自然流畅的中文重新讲述，仿佛是一位优秀的中文写作者从零开始撰写。不要有翻译腔。

2. **英式幽默的转换**：
   - "pedantic arsehole" 等自嘲表达要保留幽默感，可意译为"杠精""学究式混蛋"等
   - 保留作者的讽刺和自嘲语气，但避免中文中显得粗俗
   - 脚注中的幽默发散同样需要保留

3. **长句拆分**：英文长句要拆分为简短自然的中文句子，保持叙事节奏。

4. **文化背景处理**：
   - Deliveroo、Ocado 等品牌可保留英文，必要时加简短说明
   - British Politics Twitter 可译为"英国政治推特圈"
   - 《The Narrow Corridor》译为《窄廊》
   - Extinction Rebellion 译为"灭绝叛乱"

5. **格式保留**：所有 markdown 格式必须保留——标题层级、加粗、斜体、图片、链接、代码块、脚注引用和脚注内容。

6. **叙事节奏**：保持原文从"个人困境 → 哲学反思 → 实践行动 → 普遍启示"的叙事弧线，确保每个转折点都有足够的冲击力。

7. **准确性**：所有事实、数据和逻辑论证必须与原文完全一致，不得增删论点。
